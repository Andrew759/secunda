# Тестовое задание для Secunda

Это приложение реализовывает функционал тестового задания,
по созданию API Серивса управления задачами с командной работой и
историей изменений 

# Требования для запуска на локальном окружении
1. Должны быть установлены docker и docker compose
    1. Приложение протестировано на ОС debian

# Запуск приложения
1. В корне приложения создать файл .env, по шаблону из .env.example (файл готов к использованию на локальном окружении)
2. В корне приложения запустить ./run.sh
3. Следовать инструкциям скрипта:
    1. Запустить local go main file build - локальное построение main файла. Вынесено из контейнера для многоратного (разница до 10 порядков) ускорения билда
    2. Запустить build (no cache) либо build (with cache) - в зависимости от того, нужно ли билдить с кэшем докера или без
    3. Запустить up

# Примечание по запуску автотестов

Для запуска автотестов требуется стартовать контейнеры докер. На текущий момент используется та же БД, что и для dev
окружения. Таблицы БД удаляются после каждого теста

# Примечание по запуску миграций

1. На локальной машине должен быть установлен goose - github.com/pressly/goose
2. В папке sql проекта нужно выполнить команду goose mysql (дефолтные креды не скрыты для удобства) "user:pass@tcp(127.0.0.1:3306)/db?parseTime=true" up
3. Для отката миграций использовать команду goose mysql "user:pass@tcp(127.0.0.1:3306)/db?parseTime=true" down
4. Пароль и пользователь должен соответствовать данным из .env файла
